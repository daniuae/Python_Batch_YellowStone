# Lab 1: JDK Setup, JVM/JDK/JRE Verification, and First Run

## Objective

Install and verify the JDK, confirm JVM/JDK tools, compile and run a minimal Java program, and understand the roles of the JDK (compiler) and JVM (runtime).

---

## Step 1 — Verify Java Installation

### Where to Start
Terminal / Command Prompt

### What to Do

```bash
java -version
javac -version
```

### Why

Confirms that both:

- **java** (JVM runtime)
- **javac** (Java compiler from the JDK)

are installed and available in the system path.

### Expected Output

```text
openjdk version "21.0.2" 2024-01-16
OpenJDK Runtime Environment
OpenJDK 64-Bit Server VM

javac 21.0.2
```

---

## Step 2 — Create the Source File

### Where to Start
Any text editor (VS Code, IntelliJ IDEA, Notepad++, Vim, etc.)

### What to Do

Create a file named:

```text
HelloRuntime.java
```

Add the following code:

```java
public class HelloRuntime {
    public static void main(String[] args) {
        System.out.println("Hello from Java Runtime");
    }
}
```

### Why

This is a minimal Java program that contains:

- A class definition
- The `main()` method (application entry point)
- A statement to print output to the console

---

## Step 3 — Compile with the JDK

### Where to Start
Terminal in the directory containing `HelloRuntime.java`

### What to Do

```bash
javac HelloRuntime.java
```

### Why

The **javac** compiler converts:

```text
HelloRuntime.java
        ↓
HelloRuntime.class
```

The generated `.class` file contains **Java Bytecode**, which can be executed by any JVM.

### Verify

Check that a new file exists:

```text
HelloRuntime.class
```

---

## Step 4 — Run on the JVM

### Where to Start
Same directory as the compiled class file

### What to Do

```bash
java HelloRuntime
```

### Why

The `java` command:

1. Starts the JVM.
2. Loads the `HelloRuntime.class` bytecode.
3. Executes the `main()` method.

### Expected Output

```text
Hello from Java Runtime
```

---

## Step 5 — Inspect Bytecode with javap

### Where to Start
Terminal

### What to Do

```bash
javap -c HelloRuntime
```

### Why

Displays the bytecode instructions generated by the compiler and executed by the JVM.

### Sample Output

```text
Compiled from "HelloRuntime.java"
public class HelloRuntime {
  public static void main(java.lang.String[]);
    Code:
       0: getstatic     #2
       3: ldc           #3
       5: invokevirtual #4
       8: return
}
```

### Bytecode Explanation

| Instruction | Meaning |
|------------|----------|
| `getstatic` | Accesses `System.out` |
| `ldc` | Loads string constant |
| `invokevirtual` | Calls `println()` |
| `return` | Ends method execution |

---

## Step 6 — Confirm Tools in the JDK

### Where to Start
Terminal

### What to Do

```bash
javadoc -version
jar --version
jcmd -h
```

### Why

Demonstrates that the JDK provides more than just the JVM and compiler.

### Important JDK Tools

| Tool | Purpose |
|--------|----------|
| `javac` | Java compiler |
| `java` | JVM launcher |
| `javap` | Bytecode disassembler |
| `javadoc` | Documentation generator |
| `jar` | Package classes into JAR files |
| `jcmd` | JVM diagnostics and monitoring |
| `jconsole` | JVM monitoring GUI |
| `jps` | Lists Java processes |

# Understanding the Flow

```text
Source Code (.java)
        │
        ▼
      javac
(JDK Compiler)
        │
        ▼
Bytecode (.class)
        │
        ▼
       java
(JVM Runtime)
        │
        ▼
 Program Output
```

---

# JDK vs JRE vs JVM

| Component | Full Form | Purpose |
|------------|------------|----------|
| JVM | Java Virtual Machine | Executes bytecode |
| JRE | Java Runtime Environment | JVM + Libraries |
| JDK | Java Development Kit | JRE + Development Tools |

### Relationship

```text
JDK
 ├── JRE
 │    ├── JVM
 │    └── Core Libraries
 └── Development Tools
      ├── javac
      ├── jar
      ├── javadoc
      └── jcmd
```
